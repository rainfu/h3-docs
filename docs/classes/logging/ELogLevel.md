# ELogLevel枚举

ELogLevel枚举是VCMI日志系统中的日志级别枚举，定义了不同日志消息的严重程度。

## 枚举定义

```cpp
namespace ELogLevel
{
    enum ELogLevel
    {
        TRACE,      // 最详细的信息，通常只在开发过程中启用
        DEBUG,      // 调试信息，用于诊断问题
        INFO,       // 一般信息，确认程序按预期工作
        WARNING,    // 警告信息，表示潜在的问题
        ERROR,      // 错误信息，表示发生错误但仍可继续执行
        FATAL       // 致命错误，表示严重错误导致程序可能无法继续运行
    };

    #ifdef VCMI_ANDROID
        int toAndroid(ELogLevel logLevel); // 将日志级别转换为Android平台对应的级别
    #endif
}
```

## 功能说明

ELogLevel是VCMI日志系统中的日志级别枚举，用于区分不同日志消息的重要性和严重程度。通过设置日志级别，可以控制哪些消息被记录，哪些被忽略。较低级别的日志消息（如TRACE和DEBUG）通常在生产环境中被禁用，而较高级别的消息（如ERROR和FATAL）则始终被记录。

## 枚举值说明

- `TRACE`: 最详细的日志级别，记录最详细的程序运行信息，主要用于开发阶段调试
- `DEBUG`: 调试日志级别，记录调试相关信息，帮助开发人员诊断问题
- `INFO`: 信息日志级别，记录程序正常运行时的关键事件
- `WARNING`: 警告日志级别，记录可能存在问题但不影响程序继续运行的情况
- `ERROR`: 错误日志级别，记录错误事件，通常表示操作失败但不影响整体应用
- `FATAL`: 致命错误日志级别，记录严重的错误事件，可能导致应用终止
