# BattleSide枚举

BattleSide枚举是VCMI中战斗双方的标识，用于区分战斗中的两个对立阵营。

## 类定义

```cpp
enum class BattleSide
{
    NONE = -1,    // 无阵营（特殊值）
    ATTACKER = 0, // 攻击方
    DEFENDER = 1, // 防御方
    ALL_KNOWING   // 全知视角（用于观察者或AI）
};
```

## 功能说明

BattleSide是VCMI战斗系统中用于标识战斗双方的枚举类型。在经典的《英雄无敌》战斗系统中，通常有一方是攻击者，另一方是防御者。这个枚举用于区分这两个阵营，并提供了一些特殊值用于特殊情况，如观察者模式或全知视角。

## 枚举值

- `NONE = -1`: 无阵营，用于表示不属于任何战斗阵营的单位或状态
- `ATTACKER = 0`: 攻击方，通常是指发起攻击的一方，例如进攻城堡或攻击中立生物
- `DEFENDER = 1`: 防御方，通常是指被攻击的一方，例如守卫城堡或中立生物
- `ALL_KNOWING`: 全知视角，用于表示观察者、AI或需要了解双方信息的系统

## 设计说明

BattleSide枚举在战斗系统中发挥着基础性作用，它不仅用于区分战斗中的两个基本阵营，还提供了特殊值来处理边缘情况。在城堡攻防战中，攻击方和防御方的角色更加明显，而在野外遭遇战中，先发现对方的一方通常被视为攻击方。

特殊值ALL_KNOWING允许系统组件（如AI或UI）访问双方的信息，这对于实现智能决策和完整的游戏状态显示至关重要。NONE值则用于处理中立单位或某些特殊状态。

这个枚举的设计简洁明了，同时满足了战斗系统的基本需求和特殊情况的处理。
