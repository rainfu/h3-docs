# BattleHex::EDir枚举

BattleHex::EDir枚举是VCMI中战斗格子方向的定义，用于表示战斗场上格子之间的相对方向。

## 类定义

虽然具体的定义没有在我们已查看的文件中显示，但根据BattleHex的常见实现，我们可以推断其结构如下：

```cpp
namespace BattleHex
{
    enum class EDir
    {
        // 战斗格子的可能方向
        // 具体值取决于六边形网格的方向定义
        TOP_LEFT,     // 左上
        TOP,          // 上
        TOP_RIGHT,    // 右上
        BOTTOM_RIGHT, // 右下
        BOTTOM,       // 下
        BOTTOM_LEFT   // 左下
    };
}
```

## 功能说明

BattleHex::EDir是VCMI战斗系统中用于表示六边形格子相邻方向的枚举类型。在六角网格战斗系统中，每个格子最多有6个相邻格子，该枚举定义了这些方向。这个枚举在计算攻击范围、移动路径和邻近单位时非常重要。

## 枚举值

- `TOP_LEFT`: 左上方相邻格子方向
- `TOP`: 正上方相邻格子方向
- `TOP_RIGHT`: 右上方相邻格子方向
- `BOTTOM_RIGHT`: 右下方相邻格子方向
- `BOTTOM`: 正下方相邻格子方向
- `BOTTOM_LEFT`: 左下方相邻格子方向

## 设计说明

在六边形网格系统中，方向的定义对战斗逻辑至关重要。BattleHex::EDir枚举提供了清晰的方向表示，使代码更容易理解和维护。这些方向值在计算单位攻击范围、确定攻击角度和处理六边形网格的空间关系时发挥关键作用。

方向枚举有助于处理游戏中复杂的战斗机制，例如：
- 确定攻击者相对于目标的位置
- 计算多边形攻击的影响范围
- 判断单位是否可以攻击特定位置
- 处理特殊能力的触发方向
