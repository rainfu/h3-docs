# EWallPart枚举

EWallPart枚举是VCMI中城墙部分的定义，用于标识城堡攻防战中城墙的不同组成部分。

## 类定义

```cpp
enum class EWallPart
{
    KEEP,           // 城堡主楼
    BOTTOM_TOWER,   // 底部塔楼
    BOTTOM_WALL,    // 底部城墙
    UPPER_TOWER,    // 上部塔楼
    UPPER_WALL,     // 上部城墙
    CENTER_TOWER,   // 中央塔楼
    CENTER_GATE,    // 中央大门
    GATEHOUSE,      // 门房（大门上方的建筑）
    MOAT,           // 护城河
    BRIDGE,         // 桥梁（如果适用）
    INVALID         // 无效部分
};
```

## 功能说明

EWallPart是VCMI战斗系统中用于标识城墙各部分的枚举类型。在城堡攻防战中，城墙由多个可破坏的部分组成，每个部分都可以被独立攻击和破坏。这个枚举用于标识和管理这些不同的城墙组件，对于城堡围攻战斗的逻辑至关重要。

## 枚举值

- `KEEP`: 城堡主楼，通常是城墙结构中最坚固的部分
- `BOTTOM_TOWER`: 底部塔楼，城墙底部的防御塔
- `BOTTOM_WALL`: 底部城墙，城墙的底部段
- `UPPER_TOWER`: 上部塔楼，城墙顶部的防御塔
- `UPPER_WALL`: 上部城墙，城墙的顶部段
- `CENTER_TOWER`: 中央塔楼，城墙中央的防御塔
- `CENTER_GATE`: 中央大门，城堡的主要入口
- `GATEHOUSE`: 门房，建在大门上方的建筑结构
- `MOAT`: 护城河，围绕城堡的防御沟渠
- `BRIDGE`: 桥梁，跨越护城河的通道
- `INVALID`: 无效部分，用于表示无效或不存在的城墙部分

## 设计说明

EWallPart枚举在城堡攻防战斗系统中起着关键作用，它允许游戏精确跟踪城墙各部分的状态。每个城墙部分可能有不同的生命值、防御力和特殊属性，玩家可以有选择地攻击特定部分。这种设计模拟了真实城堡攻防战的复杂性，增加了战斗策略的深度。

城墙各部分之间可能存在依赖关系，例如，当大门被摧毁后，门房可能也会受到影响。通过使用枚举类型，VCMI能够以类型安全的方式处理这些复杂的交互逻辑。
